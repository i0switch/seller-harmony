CAND-P1-02 Discordロール付与の信頼性向上と状態可視化
Discord連携後、DB上のステータスが active にもかかわらずロールが正常に付与されない問題（CAND-P1-02）を包括的に修正しました。 バックエンドでの自動付与ロジックの強化、role_assignments テーブルによる正確な状態追跡、およびフロントエンドでのリトライ導線の整備を完了しています。

実施内容
1. バックエンド: 自動ロール付与の網羅的実装
OAuth完了時: Discord OAuth完了直後、保留中 (pending_discord) のメンバーシップを即座にアクティベートし、自動でロール付与を実行します。2段階認証フロー（確認後の保存）にも対応しました。
支払い復帰時: Stripe Webhook (invoice.payment_succeeded) で支払い失敗（grace_period 等）から active に復帰した際、自動的にロールを再付与するロジックを追加しました。
エラー処理の共通化: 

assignDiscordRole
 ユーティリティを強化し、成否とエラー原因を role_assignments に記録します。
2. バックエンド: リトライ導線の堅牢化 (discord-bot)
ユーザーの手動リトライ時に呼び出される discord-bot 関数において、role_assignments への記録を insert から upsert に変更しました。これにより、リトライ回数に関わらずユニーク制約エラーなしで最新状態を記録できます。
3. フロントエンド: 推測ではなく実態に基づく状態表示 (buyerApi)
buyerApi.getMemberships において role_assignments テーブルを結合参照するように修正。
画面上の表示（「付与済」「失敗」「保留中」）が、DB内の実際の Discord API 実行結果と1対1で対応するようになりました。
4. インフラ: データベース整合性の確保
role_assignments の membership_id にユニーク制約を追加。
