### 失敗した場合

| 症状 | 原因 | 対処 |
|---|---|---|
| 「Invalid login credentials」 | メール/パスワードが違う | Supabase Auth → Users でアカウントを確認 |
| 「Email not confirmed」 | メール未確認 | Supabase Auth → Users → 該当ユーザー → Confirm email |
| リダイレクトされない | ルーティング設定の問題 | コンソールログを確認 |
| `/seller` にリダイレクト | ロールが seller になっている | Supabase → users テーブル → role を `platform_admin` に変更 |

## Task 3: 管理者ダッシュボードの表示確認

### 手順

1. ダッシュボード `/platform` が表示されていることを確認

2. 以下の画面要素を確認:
   - **サイドバーメニュー**:
     - ダッシュボード
     - テナント管理
     - 審査待ち
     - システムお知らせ
   - **ヘッダー**: ログインユーザー名またはロゴ
   - **メインコンテンツ**: 各種統計またはリスト

### 期待される結果

- ダッシュボードが正常に表示される
- 統計データが表示される（Supabase からの実データ）
  - テナント数（0件でもOK — 初期状態）
  - 審査待ち件数（0件でもOK）
  - 月間売上（¥0でもOK）
