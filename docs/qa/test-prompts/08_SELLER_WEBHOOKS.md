# テスト08: Seller Webhook履歴

> **カテゴリ**: Webhook監視  
> **優先度**: P2 (Medium)  
> **推定所要時間**: 15分  
> **前提条件**: Sellerログイン済み

---

## AIエージェントへの指示

```
SellerのWebhook履歴ページを検証してください。
テスト環境: https://preview--member-bridge-flow.lovable.app/seller/webhooks
フィルタ、詳細ダイアログ、再処理機能を確認してください。
```

---

## TC-08-01: Webhook一覧ページの表示

**手順**:
1. `/seller/webhooks` にアクセス

**期待結果**:
- [ ] 見出し「Webhook履歴」が表示される
- [ ] フィルターセレクト（「すべて」がデフォルト）
- [ ] Webhookイベントカードが一覧表示される
- [ ] 各カードに以下が表示される:
  - イベントタイプ（font-mono、例: `checkout.session.completed`）
  - ステータスバッジ（「成功」「失敗」「処理中」）
  - 署名検証: 「✓」または「✗」
  - 受信日時

---

## TC-08-02: フィルタリング

**手順（各フィルターを順番に選択）**:
1. 「成功」→ 成功のイベントのみ表示
2. 「失敗」→ 失敗のイベントのみ表示
3. 「処理中」→ 処理中のイベントのみ表示
4. 「すべて」→ 全イベント表示

**期待結果**:
- [ ] 各フィルターで正しいイベントが表示される
- [ ] 該当なしの場合「Webhookイベントがありません」

---

## TC-08-03: 詳細ダイアログ

**手順**:
1. 任意のイベントのEye（詳細）アイコンをクリック

**期待結果**:
- [ ] ダイアログが表示される
- [ ] タイトル: イベントタイプ
- [ ] Description: 「Webhook詳細」
- [ ] 以下の情報が表示される:
  - 「Event ID」
  - 「ステータス」（バッジ）
  - 「署名」（「✓ 検証済み」or「✗ 検証失敗」）
  - 「受信日時」
  - エラー情報（失敗イベントの場合）
  - 「ペイロード」（`<pre>`タグでJSON表示）

**手順**:
2. ダイアログを閉じる（X ボタン or 背景クリック）

**期待結果**:
- [ ] ダイアログが閉じる

---

## TC-08-04: 失敗イベントの再処理

**手順**:
1. 「失敗」ステータスのイベントのRefreshCw（再処理）アイコンをクリック

**期待結果**:
- [ ] ConfirmDialog:
  - タイトル: 「再処理しますか？」
  - 説明: 「{eventType} を再処理します。」
  - ボタン: 「キャンセル」「再処理」

**手順**:
2. 「再処理」をクリック

**期待結果**:
- [ ] Toast: 「再処理を開始」「{eventType} の再処理を開始しました」

---

## TC-08-05: 成功イベントには再処理ボタンがないこと

**手順**:
1. 「成功」ステータスのイベントを確認

**期待結果**:
- [ ] RefreshCw（再処理）ボタンが表示されない（詳細ボタンのみ）

---

## TC-08-06: 署名検証失敗イベントの表示

**手順**:
1. 署名検証失敗（`signatureVerified: false`）のイベントを確認

**期待結果**:
- [ ] 「✗」が表示される
- [ ] 詳細ダイアログで「✗ 検証失敗」が表示される
