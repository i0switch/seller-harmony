- [ ] 検索バー（placeholder: 「テナント名またはメールで検索...」）
- [ ] フィルターセレクト（「すべて」がデフォルト）
- [ ] テーブルに以下の列:

  - 「Stripe」（stripeStatusLabel）
  - 「会員数」（ソート可能、右寄せ）
  - 「エラー」（ソート可能、右寄せ、>0でバッジ）
  - 「最終アクティブ」（ソート可能、右寄せ）
  - アクション列

## TC-12-03: テナントフィルタリング

**手順（順番に選択）**:
1. 「契約中」→ status=active のみ
2. 「試用中」→ status=trial のみ  
3. 「停止中」→ status=suspended のみ
4. 「解約済」→ status=canceled のみ
5. 「すべて」→ 全テナント

**期待結果**:
- [ ] 各フィルターで正しいテナントのみが表示される
- [ ] フィルターの切り替えが即座に反映される

## TC-12-04: テナントテーブルのソート

**手順**:

2. 「会員数」列ヘッダーをクリック
3. 「エラー」列ヘッダーをクリック
4. 「最終アクティブ」列ヘッダーをクリック

**期待結果**:
- [ ] 各列でソートが切り替わる（昇順↔降順）
- [ ] ソートインジケーターが表示される
- [ ] データの順序が正しく変わる

## TC-12-08: テナント停止（一覧から）

**手順**:
1. status=activeのテナントのアクション列で「停止」ボタンをクリック

**期待結果**:
- [ ] ConfirmDialog:
  - 「テナントを停止しますか？」
  - 「{name} のサービスを停止します。会員への影響がありますがよろしいですか？」
  - 「キャンセル」「停止する」（destructive）

**手順**:
2. 「停止する」をクリック

**期待結果**:
- [ ] Toast: 「テナント停止」「{name} を停止しました（モック）」
- [ ] テナントのステータスバッジが「停止中」に変わる
