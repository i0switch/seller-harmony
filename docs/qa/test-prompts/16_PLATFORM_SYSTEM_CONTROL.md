## TC-16-01: システム制御ページの表示

**手順**:

**期待結果**:
- [ ] 見出し「システム制御（Kill Switch）」（Shieldアイコン付き）
- [ ] ステータスバナー:
  - 有効なスイッチ > 0: 「{N}個のKill Switchが有効です」「一部の自動処理が停止されています」
  - 全スイッチ無効: 「すべて正常稼働中」「すべての自動処理が有効です」
- [ ] 各Kill Switchカードが表示される（4つ）

---

## TC-16-02: Kill Switchカードの表示

**手順**:
1. 各Kill Switchカードの内容を確認

**期待結果**:
- [ ] 各カードに以下が表示:
  - スイッチ名
  - ステータスバッジ: 「停止中」（destructive/赤）or「稼働中」（secondary）
  - 説明テキスト
  - 「最終変更: {日時} by {ユーザー}」
  - Switchトグル

---

## TC-16-03: Kill Switch — 停止操作（稼働中→停止）

**手順**:
1. 「稼働中」のKill Switchのトグルをクリック

**期待結果**:
- [ ] AlertDialog:
  - タイトル: 「⚠️ 自動処理を停止しますか？」
  - 説明: 「{スイッチ名} を停止します。この操作により関連する自動処理がすべて停止されます。」
  - 「キャンセル」「停止する」（destructive/赤）

**手順**:
2. 「キャンセル」をクリック

**期待結果**:
- [ ] ダイアログが閉じる
- [ ] トグルの状態は変わらない（稼働中のまま）

**手順**:
3. 再度トグルクリック→「停止する」をクリック

**期待結果**:
- [ ] Toast: 「⚠️ 自動処理を停止しました」「{スイッチ名} を停止しました」
- [ ] バッジが「停止中」に変わる
- [ ] トグルがON（停止状態=enabled=true）になる
- [ ] ステータスバナーの有効数が更新される

---

## TC-16-04: Kill Switch — 再開操作（停止→稼働中）

**手順**:
1. 「停止中」のKill Switchのトグルをクリック

**期待結果**:
- [ ] AlertDialog:
  - タイトル: 「自動処理を再開しますか？」
  - 説明: 「{スイッチ名} を再開します。」
  - 「キャンセル」「再開する」

**手順**:
2. 「再開する」をクリック

**期待結果**:
- [ ] Toast: 「✅ 自動処理を再開しました」「{スイッチ名} を再開しました」
- [ ] バッジが「稼働中」に変わる
- [ ] ステータスバナーの有効数が更新される
