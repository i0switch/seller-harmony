# テスト05: Seller プラン管理（CRUD・公開/停止）

> **カテゴリ**: プラン管理  
> **優先度**: P0 (Critical)  
> **推定所要時間**: 25分  
> **前提条件**: Sellerログイン済み、オンボーディング完了

---

## AIエージェントへの指示

```
Sellerのプラン管理機能を網羅的に検証してください。
テスト環境: https://preview--member-bridge-flow.lovable.app/seller/plans
プラン一覧の表示、フィルタリング、新規作成、編集、公開/停止トグル、
バリデーション、Discord検証を確認してください。
```

---

## TC-05-01: プラン一覧ページの表示

**手順**:
1. `/seller/plans` にアクセス

**期待結果**:
- [ ] 見出し「プラン管理」が表示される
- [ ] 「新規プラン」ボタン（Plusアイコン付き）が表示される
- [ ] フィルターセレクトがある（「すべて」がデフォルト）
- [ ] プランカードが一覧表示される
- [ ] 各カードに以下が表示される:
  - プラン名
  - ステータスバッジ（「公開中」「停止」「下書き」）
  - 種別バッジ（「月額」「単発」）
  - 価格（¥形式 + 「/ 月」または「単発」）
  - 会員数
  - ロール情報（ロールID or 「指定なし」）
  - 権限付与期限（該当する場合）
- [ ] 「編集」ボタンと「公開/停止」ボタンが各カードにある

---

## TC-05-02: プランフィルタリング

**手順**:
1. フィルターで「公開中」を選択

**期待結果**:
- [ ] ステータスが「公開中」のプランのみ表示される

**手順**:
2. フィルターで「停止」を選択

**期待結果**:
- [ ] ステータスが「停止」のプランのみ表示される

**手順**:
3. フィルターで「下書き」を選択

**期待結果**:
- [ ] ステータスが「下書き」のプランのみ表示される

**手順**:
4. フィルターで「すべて」を選択

**期待結果**:
- [ ] すべてのプランが表示される

**手順**:
5. 該当するプランが0件のフィルターを選択

**期待結果**:
- [ ] 「条件に一致するプランがありません。」の空状態メッセージ

---

## TC-05-03: 新規プラン作成ページへの遷移

**手順**:
1. 「新規プラン」ボタンをクリック

**期待結果**:
- [ ] `/seller/plans/new` に遷移する
- [ ] 見出し「新規プラン作成」が表示される
- [ ] 「プラン一覧」戻りリンクが表示される

---

## TC-05-04: 新規プラン作成 — フォーム要素の確認

**手順**:
1. `/seller/plans/new` のフォーム要素を確認

**期待結果**:
- [ ] 「プラン名 *」入力欄（placeholder: 「例: プレミアム会員」）
- [ ] 「説明」テキストエリア（placeholder: 「プランの説明...」）
- [ ] 「種別」セレクト: 「月額（サブスク）」「単発」
- [ ] 「金額（JPY） *」入力欄（placeholder: 「980」）
- [ ] 「DiscordサーバーID *」入力欄（placeholder: 「サーバーID」）
- [ ] 「DiscordロールID *」入力欄（placeholder: 「ロールID」）
- [ ] 「Discord設定を検証」ボタン
- [ ] 「公開する」スイッチ
- [ ] 「キャンセル」ボタンと「作成」ボタン

---

## TC-05-05: 新規プラン作成 — バリデーション（空送信）

**手順**:
1. すべてのフィールドを空にして「作成」をクリック

**期待結果**:
- [ ] 「プラン名を入力してください」エラー
- [ ] 「有効な金額を入力してください」エラー
- [ ] 「DiscordサーバーIDを入力してください」エラー
- [ ] 「DiscordロールIDを入力してください」エラー
- [ ] フォームが送信されない

---

## TC-05-06: 新規プラン作成 — 月額プランの作成

**手順**:
1. プラン名: 「テストプレミアム」
2. 説明: 「テスト用のプレミアムプラン」
3. 種別: 「月額（サブスク）」
4. 金額: 「2980」
5. DiscordサーバーID: 「1234567890123456789」
6. DiscordロールID: 「9876543210987654321」
7. 「作成」をクリック

**期待結果**:
- [ ] バリデーションエラーなし
- [ ] ボタンがローディング状態になる
- [ ] 成功Toast: 「保存しました」「プラン「テストプレミアム」を保存しました。」
- [ ] `/seller/plans` にリダイレクト
- [ ] 一覧に新しいプランが表示される

---

## TC-05-07: 新規プラン作成 — 単発プランの作成

**手順**:
1. 「新規プラン」をクリック
2. プラン名: 「単発アクセス」
3. 種別: 「単発」を選択

**期待結果**:
- [ ] 種別を「単発」に切り替えると、追加フィールドが表示される:
  - 「権限付与期間ポリシー」セレクト: 「無期限」「期限付き」
- [ ] 「期限付き」を選択すると「付与日数」入力欄（placeholder: 「30」）が表示される

**手順**:
4. 権限付与期間: 「期限付き」
5. 付与日数: 「30」
6. 金額: 「500」
7. Discord設定を入力
8. 「作成」をクリック

**期待結果**:
- [ ] 成功Toast表示
- [ ] 一覧にリダイレクト

---

## TC-05-08: 単発プラン — 付与日数バリデーション

**手順**:
1. 種別「単発」、権限付与期間「期限付き」で付与日数を空にして作成

**期待結果**:
- [ ] 「有効な日数を入力してください」エラー
- [ ] 付与日数に「0」や負の値を入力した場合もエラー

---

## TC-05-09: Discord検証ボタン

**手順**:
1. サーバーIDとロールIDを入力
2. 「Discord設定を検証」をクリック

**期待結果**:
- [ ] ボタンが「検証中」に変わる
- [ ] 検証完了後:
  - OK: `data-testid="discord-ok"` 要素に「検証OK」テキスト
  - NG: `data-testid="discord-error"` 要素に「検証NG - サーバーIDとロールIDを確認してください」

---

## TC-05-10: 公開スイッチとDiscord検証警告

**手順**:
1. Discord検証を実行せずに「公開する」スイッチをONにする

**期待結果**:
- [ ] 警告メッセージ「Discord設定の検証が未完了またはエラーです。公開前に検証してください。」が表示される

**手順**:
2. Discord検証を実行してOKになった後、「公開する」をONにする

**期待結果**:
- [ ] 警告が表示されない

---

## TC-05-11: プラン編集ページ

**手順**:
1. プラン一覧で任意のプランの「編集」ボタンをクリック

**期待結果**:
- [ ] `/seller/plans/{id}` に遷移する
- [ ] 見出し「{プラン名} を編集」が表示される
- [ ] 既存のデータがフォームにプリセットされている
- [ ] 会員数が1名以上の場合、警告「このプランにはX名の会員がいます。価格変更は既存会員には適用されません。」

---

## TC-05-12: プラン編集 — 保存

**手順**:
1. プラン名を変更（例: 「テスト編集後プラン」）
2. 「保存」をクリック

**期待結果**:
- [ ] 成功Toast: 「保存しました」
- [ ] `/seller/plans` にリダイレクト
- [ ] 一覧で更新後のプラン名が表示される

---

## TC-05-13: プラン公開/停止トグル

**手順**:
1. 「公開中」のプランの「停止」ボタン（EyeOffアイコン）をクリック

**期待結果**:
- [ ] AlertDialog表示:
  - タイトル: 「プランを停止しますか？」
  - 説明: 「新規購入ができなくなります。既存会員には影響しません。」
  - ボタン: 「キャンセル」「停止する」

**手順**:
2. 「キャンセル」をクリック

**期待結果**:
- [ ] ダイアログが閉じる
- [ ] プランのステータスは変わらない

**手順**:
3. 再度「停止」ボタンをクリック→「停止する」をクリック

**期待結果**:
- [ ] プランのステータスバッジが「停止」に変わる

---

## TC-05-14: プラン停止→公開トグル

**手順**:
1. 「停止」中のプランの「公開」ボタン（Eyeアイコン）をクリック

**期待結果**:
- [ ] AlertDialog:
  - タイトル: 「プランを公開しますか？」
  - 説明: 「このプランが購入可能になります。」
  - ボタン: 「キャンセル」「公開する」
  
**手順**:
2. 「公開する」をクリック

**期待結果**:
- [ ] プランのステータスバッジが「公開中」に変わる

---

## TC-05-15: 「キャンセル」ボタン

**手順**:
1. プラン作成・編集画面で「キャンセル」をクリック

**期待結果**:
- [ ] `/seller/plans` にリダイレクトされる
