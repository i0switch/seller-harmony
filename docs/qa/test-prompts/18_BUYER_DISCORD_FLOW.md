# テスト18: Buyer Discord連携フロー（確認→OAuth→結果）

> **カテゴリ**: Discord連携  
> **優先度**: P0 (Critical)  
> **推定所要時間**: 20分  
> **前提条件**: 決済完了ページからの遷移、またはBuyer認証状態

---

## AIエージェントへの指示

```
Buyer（購入者）のDiscord連携フロー全体を検証してください。
テスト環境: https://preview--member-bridge-flow.lovable.app/
3ステップのフロー:
  1. /buyer/discord/confirm — アカウント確認
  2. Discord OAuth2 (外部/モック)
  3. /buyer/discord/result — OAuth結果処理

CSRF state検証、誤連携防止UX、エラーハンドリングを重点的に確認してください。
```

---

## TC-18-01: Discord連携確認ページの表示

**手順**:
1. `/buyer/discord/confirm` にアクセス

**期待結果**:
- [ ] 見出し「Discord連携の確認」
- [ ] サブテキスト「以下のDiscordアカウントで連携します。正しいか確認してください。」
- [ ] ユーザー情報:
  - Discordユーザー名
  - Discord ID
  - バッジ: 「OAuth認証済み」
- [ ] スマホ向けAlert:
  - 「スマホの方へ：ブラウザでログイン中のDiscordアカウントが表示されます。普段と違うアカウントの場合は、「別のアカウントで連携する」を選んでください。」
- [ ] ボタン:
  - 「このアカウントで連携する」
  - 「別のアカウントで連携する」（RotateCcwアイコン）
- [ ] 連携説明:
  - 「連携すると以下が行われます：」
  - 「Discordアカウントの認証情報の確認」
  - 「ファンクラブサーバーへの自動参加」
  - 「購入プランに応じたロールの付与」

## TC-18-02: 「このアカウントで連携する」ボタン

**手順**:
1. 「このアカウントで連携する」をクリック

**期待結果**:
- [ ] ボタンが「連携中...」に変わる
- [ ] discord-oauth Edge Function が呼び出される
- [ ] 成功の場合: Discord OAuth フローが開始される

## TC-18-03: 「別のアカウントで連携する」ボタン

**手順**:

**期待結果**:
- [ ] Discord OAuth再認証フローが開始される（別のアカウントで再ログインを促す）

## TC-18-06: Discord結果ページ — 連携完了

**手順**:

**期待結果**:

- [ ] 詳細情報:

  - 「プラン」— プラン名
