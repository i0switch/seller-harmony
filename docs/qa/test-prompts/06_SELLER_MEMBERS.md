## TC-06-01: 会員一覧ページの表示

**手順**:
1. `/seller/members` にアクセス

**期待結果**:
- [ ] 見出し「会員管理」が表示される
- [ ] 検索バー（placeholder: 「名前・メール・Discordで検索...」）
- [ ] フィルターセレクト（「すべて」がデフォルト）
- [ ] テーブルが表示され、以下の列がある:
  - 「会員」（名前 + メール、ソート可能）
  - 「プラン」
  - 「課金」（ステータスバッジ）
  - 「Discord」（ユーザー名 or 「未連携」）
  - 「ロール」（ステータスバッジ）
  - 「エラー」（「あり」 or 「—」）
  - アクション列: 「詳細」ボタン
- [ ] ページネーションが表示される（5件/ページ）

---

## TC-06-02: 会員検索

**手順**:
1. 検索バーにモック会員の名前の一部を入力（例: 「太郎」）

**期待結果**:
- [ ] 名前に「太郎」を含む会員のみ表示される

**手順**:
2. 検索バーにメールアドレスの一部を入力

**期待結果**:
- [ ] メールアドレスがマッチする会員が表示される

**手順**:
3. 検索バーにDiscordユーザー名の一部を入力

**期待結果**:
- [ ] Discordユーザー名がマッチする会員が表示される

**手順**:
4. 存在しない名前で検索

**期待結果**:
- [ ] テーブルが空になり「会員がいません」の空状態メッセージ

---

## TC-06-03: 会員フィルタリング

**手順**:
1. フィルターで「有効」を選択

**期待結果**:
- [ ] 課金状態が「active」の会員のみ表示

**手順**:
2. フィルターで「支払い遅延」を選択

**期待結果**:
- [ ] 課金状態が「grace_period」の会員のみ表示

**手順**:
3. フィルターで「解約済」を選択

**期待結果**:
- [ ] 課金状態が「canceled」「expired」の会員が表示

**手順**:
4. フィルターで「未払い」を選択

**期待結果**:
- [ ] 課金状態が「payment_failed」の会員が表示

---

## TC-06-04: 会員テーブルのソート

**手順**:
1. 「会員」列ヘッダーをクリック

**期待結果**:
- [ ] 名前の昇順でソートされる
- [ ] ソートインジケーター（↑）が表示される

**手順**:
2. 再度「会員」列ヘッダーをクリック

**期待結果**:
- [ ] 名前の降順でソートされる
- [ ] ソートインジケーター（↓）が表示される

## TC-06-05: ページネーション

**手順**:
1. 10件の会員データで5件/ページのページネーションを確認
2. 「次のページ」ボタンをクリック

**期待結果**:
- [ ] 2ページ目の会員が表示される（6-10件目）
- [ ] 1ページ目とは異なる会員が表示される
- [ ] ページ番号表示が更新される（例: 「2/2ページ」）
- [ ] 全件数が表示される（例: 「全10件」）

**手順**:
3. 「前のページ」ボタンをクリック

**期待結果**:
- [ ] 1ページ目に戻る

## TC-06-08: 会員詳細 — 再付与アクション

**手順**:
1. 「再付与」ボタンをクリック

**期待結果**:
- [ ] ConfirmDialog表示:
  - タイトル: 「ロールを再付与しますか？」
  - 説明: 「{会員名} にDiscordロールを再付与します。」
  - ボタン: 「キャンセル」「再付与」

**手順**:
2. 「キャンセル」をクリック

**期待結果**:
- [ ] ダイアログが閉じる

**手順**:
3. 再度「再付与」→ 「再付与」ボタンをクリック

**期待結果**:
- [ ] Toast: 「再付与しました」「{会員名} に対して再付与を実行しました（モック）」
