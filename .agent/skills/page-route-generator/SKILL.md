# Page & Route Generator

> 💡 **page-route-generator スキルを読み込みました**  
> 新しいページを作成し、ルーティング設定を自動更新します。

## When to Use

- 「新しい画面を追加したい」「〇〇ページを作って」と言われたとき
- アプリケーションに新しい機能の入り口（ルート）を設けるとき

## 手順

### Step 1: ページ要件の確認
- ページの目的、URLのパス（例: `/dashboard`, `/users/:id` など）をユーザーに確認する。
- ページ内に配置する主要なコンポーネントやレイアウト構成を把握する。

### Step 2: ページコンポーネントの作成
- `src/pages/` （ディレクトリが存在しなければ作成する）配下に、新しいページコンポーネント（例: `DashboardPage.tsx`）を生成する。
- ページコンポーネントのひな形には、全体を囲う適切な `div` タグとコンテナクラスを含める。

### Step 3: ルーティングへの組み込み
- `src/App.tsx` またはルーティングの定義ファイル（例: `src/routes.tsx` や `src/main.tsx` の RouterProvider 部分）を開く。
- 新しく作成したページコンポーネントを import する。
- React Router v6 の仕様に従い、`<Route path="..." element={<PageName />} />` あるいはオブジェクト形式ベースでルート宣言を追加する。

### Step 4: ブラウザでの確認
- 開発サーバーにて、作成したルート（パス）へアクセスして正しく画面が表示されるか確認するようユーザーに伝える。
