Discord OAuth Link Collision (C07) Fix Walkthrough
This fix addresses the issue where users receive a generic error when attempting to link a Discord account that is already associated with another user account.

Changes Made
Backend: Supabase Edge Function
File: 

index.ts
Modification: Wrapped the discord_identities table upsert in a try-catch block.
Improved Logic: Specifically catches Postgres error 23505 (unique_violation) and returns a JSON response with error code DISCORD_ALREADY_LINKED and HTTP status 409 Conflict.
Frontend: Buyer UI
File: 

DiscordResult.tsx
New Recovery UI:
Detects DISCORD_ALREADY_LINKED error code.
Displays a specialized "Recovery Hint" section.
Suggests the user might have logged in with a different email (e.g., Google OAuth).
Provides a direct link to the Auth page to switch accounts.
Improved Error Handling: Captures error codes from Supabase Function responses and maps them to appropriate UI states.
Verification Results
Backend Error Handling
The upsert logic was reinforced to properly catch database-level conflicts, which previously resulted in a generic "Internal server error".

Frontend UX
The failure screen now distinguishes between general OAuth failures and account link collisions, reducing support burden by guiding the user to self-recover.

NOTE

Pre-existing lint errors in the Edge Function related to Deno globals and ESM imports were observed but left untouched as they are specific to the Deno runtime environment and do not affect functional correctness in production.
